## Why I built this

I had a family member go through a cervical cancer scare a couple years back. The whole thing made me realize how much early detection matters — caught early, it's highly treatable. Caught late, it's not. When I was looking for a project that combined ML with something actually meaningful, this felt like the obvious choice.

I'm not claiming this replaces a doctor. It doesn't. But I wanted to understand what the data actually says — which risk factors matter most, how well off-the-shelf ML models can do on real clinical data, and what explainability looks like in a medical context. Turns out SHAP values are genuinely useful here, not just a nice-to-have.

---

## Dataset

Using the **UCI ML Repository — Cervical Cancer (Risk Factors) Dataset** collected at Hospital Universitario de Caracas in Venezuela.

- 858 patients, 36 features
- Target variable: biopsy result (positive/negative)
- Heavy class imbalance (~93% negative) — this trips up a lot of models if you're not careful
- Lots of missing values, especially in STD-related fields (patients often declined to answer)

The dataset is available at: https://archive.ics.uci.edu/ml/datasets/Cervical+cancer+%28Risk+Factors%29

Key features used:
- Age, number of sexual partners, age at first sexual intercourse
- Number of pregnancies
- Smoking habits and duration
- Hormonal contraceptive use and duration
- IUD use and duration
- STD history and count
- Prior cancer diagnoses

---

## What I tried and what actually worked

### Preprocessing

Missing values are recorded as `?` in the raw CSV, so I replaced those with NaN first. Then used median imputation for numerical columns. Tried mean imputation too — didn't make much of a difference honestly.

The class imbalance was the bigger problem. About 93% of samples are negative. If you just train without handling this, the model learns to predict negative for everything and gets 93% accuracy, which is completely useless. I used `class_weight='balanced'` for sklearn models and `scale_pos_weight` for XGBoost.

### Model Results

| Model | Accuracy | F1 Score | ROC-AUC |
|-------|----------|----------|---------|
| Logistic Regression | 83.7% | 0.412 | 0.847 |
| Random Forest | 88.4% | 0.531 | 0.891 |
| **XGBoost** | **91.3%** | **0.618** | **0.923** |

XGBoost won pretty clearly. The F1 score is the metric I care about most here — accuracy is misleading with class imbalance. An F1 of 0.618 isn't amazing but it's honest: the dataset is genuinely hard and small.

Logistic Regression was actually decent for a baseline, but it struggled with the interaction effects between features (like smoking duration × STD count).

## SHAP Feature Importance

I used SHAP (SHapley Additive exPlanations) to understand which features actually drive predictions. This matters clinically — a black-box model isn't useful if a doctor can't understand why it flagged someone.

![SHAP Summary Plot](results/shap_summary.png)

Top features by mean |SHAP value|:

1. **Number of sexual partners** — highest impact by a wide margin
2. **Age at first sexual intercourse** — younger age = higher long-term risk, which matches clinical literature
3. **STD count** — presence of multiple STDs strongly pushes toward positive prediction
4. **Hormonal contraceptive years** — longer duration correlates with higher risk
5. **Number of pregnancies** — moderate positive correlation with risk

The beeswarm plot (in `results/`) shows that smoking years and IUD use have more complex, non-linear effects. Moderate IUD use actually seems slightly protective in this dataset, which I found surprising.

## Known Limitations

- **Small dataset**: 858 rows is not a lot. The model would likely perform differently on data from a different hospital/region.
- **Self-reported data**: STD history and sexual behavior are self-reported. There's probably underreporting, especially for sensitive questions.
- **Class imbalance**: Even with balancing tricks, the model is still not great at catching true positives. In a medical setting, false negatives are the dangerous ones.
- **No temporal data**: Some features like "STDs: Time since first diagnosis" have 87%+ missing values — I dropped these. They're probably clinically important.
- **Not validated on external data**: I haven't tested this on any other cervical cancer dataset. It might overfit to quirks in this specific hospital's population.

---

## What I'd Do Differently

- Try SMOTE or ADASYN for oversampling instead of just class weighting — I started on this but ran into issues with the missing value imputation pipeline order, and ran out of time
- Get more data somehow. 858 rows is just not enough for a medical model
- Spend more time on the STD-related features — there's probably signal there but the missing data made it hard to use
- Try a neural net with uncertainty estimation (like MC dropout) so the model can say "I'm not sure" instead of always giving a confident answer
- Add a proper train/val/test split instead of just train/test — I was worried about having too little data in each split
